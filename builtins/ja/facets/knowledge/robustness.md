# 堅牢性知識

堅牢性は原子性や部分成功への好みではなく、操作に指定された結果から判定する。まず正常系を読み、確定する副作用と契約境界を特定する。

## 失敗時結果の選定

失敗を判定する前に、必要な結果を元要件、仕様、既存契約から定める。

| 基準 | 判定 |
|------|------|
| 失敗時の振る舞いを現行実装だけから推測している | REJECT |
| 後段のサブステップが失敗し得るだけで部分結果を許容している | REJECT |
| 原子性または部分成功が元要件、仕様、既存契約から導かれる | OK |
| 必要な結果が本当に未規定であり、勝手に作らず曖昧さとして報告する | OK |

## 失敗・再試行・中断経路

非正常経路はすべて、正常系で確定する副作用と外部から見える結果に対して比較する。

| 基準 | 判定 |
|------|------|
| 失敗で未報告の確定副作用、再試行時の重複、副作用の未解放が残る | REJECT |
| 指定された防御なしに、再試行が冪等でない外部副作用を繰り返せる | REJECT |
| 中断により必須の後始末、補償、永続状態遷移を飛ばす | REJECT |
| 失敗、再試行、中断、後始末が指定結果と観測可能な契約を保つ | OK |

## 部分成功の契約

部分成功は、呼び出し側が識別でき、仕様が許可するときだけ契約になる。

| 基準 | 判定 |
|------|------|
| 一部の副作用が残るのに、結果が全成功または全失敗として報告される | REJECT |
| 部分完了が見えるのに、継続や補償に必要な情報を呼び出し側が得られない | REJECT |
| 仕様が部分完了と、その結果、再試行、補償の意味を定めている | OK |

## 入力上限と部分結果の可観測性

外部入力の hard cap は、入力量に比例する buffer、decode、parse、展開、write より前、またはストリーミング中に適用される。部分成功や skip は、失敗項目、継続可否、部分結果を呼び出し側または利用者が識別できるときだけ観測可能な結果になる。

| 基準 | 判定 |
|------|------|
| 全量取得後にだけ size check し、入力量に比例するメモリ使用を許す | REJECT |
| hard cap を入力比例の処理より前、またはストリーミング中に強制する | OK |
| metadata または上流保証だけに依存し、一次契約でその正しさを証明できない | REJECT |
| skip や部分失敗があっても、全成功に見えるか失敗項目を識別できない | REJECT |
| 呼び出し側または利用者が失敗項目、継続可否、部分結果を観測できる | OK |
