# 失敗境界知識

## 必須処理と任意処理

失敗境界は、操作の必要性と、失敗後に保持すべき結果から決まる。

| 基準 | 判定 |
|------|------|
| 必須処理の失敗後も成功として継続する | REJECT |
| 任意処理の失敗が指定された主結果まで失敗させる | REJECT |
| 任意処理の失敗を識別可能に記録し、主結果を保持する | OK |
| 仕様にない部分成功を推測で導入する | REJECT |

## 失敗の伝播と可視性

回復可能な失敗は、捕捉、分類、集約、caller または利用者への可視化が同じ意味を保つ必要がある。

| 基準 | 判定 |
|------|------|
| 回復可能な失敗が未捕捉で上位へ伝播する | REJECT |
| 継続するが失敗項目や警告が観測できない | REJECT |
| 捕捉した失敗と保持した部分結果を caller が識別できる | OK |
| 失敗の捕捉、通知、結果保持のいずれかを証跡で確認できない | 未証明 |

## 正常系の欠落との区別

正常終了した経路で値が欠落する問題は、失敗伝播や継続判断ではない。

| 観測 | 分類 |
|------|------|
| 正常保存時に値を渡さない | 値の配線・永続化 |
| 任意処理の例外で主結果が返らない | 失敗境界 |
| 取得済み資源が解放範囲から漏れる | 資源所有権 |
