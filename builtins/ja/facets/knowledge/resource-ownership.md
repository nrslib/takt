# 資源所有権知識

## 所有権の連鎖

資源の寿命は、取得時の owner、明示的な所有権移譲、last consumer、解放責務の連鎖で決まる。

| 基準 | 判定 |
|------|------|
| 取得後の owner が不明 | REJECT |
| 所有権移譲後も元 owner と移譲先の両方が解放し得る | REJECT |
| last consumer より前に解放する | REJECT |
| 単一の owner が last consumer 後の解放を保証する | OK |

## 解放範囲

解放処理が存在しても、取得後のすべての経路が保護範囲へ入らなければ資源寿命は保証されない。

| 基準 | 判定 |
|------|------|
| 取得が `try` より前にあり、その後の失敗で `finally` に入らない | REJECT |
| 早期終了、失敗、中断、再試行のいずれかが解放を迂回する | REJECT |
| 取得成功後の全経路が同じ解放責務へ収束する | OK |
| 取得自体が失敗し、解放対象が生成されていない | 対象外 |

## 値と資源の区別

保存する値と、明示的な解放を必要とする資源は異なる。値の欠落だけでは資源リークを証明しない。

| 観測 | 分類 |
|------|------|
| 保存時に値を空値へ置き換える | 値の配線・永続化 |
| 取得済み資源を last consumer 後も解放しない | 資源所有権 |
| 任意処理の例外が主結果を失敗させる | 失敗境界 |
