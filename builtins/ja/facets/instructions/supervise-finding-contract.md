実行済みのテスト・ビルド・動作確認の証跡を確認し、Finding Contract の最終承認を行ってください。

手順:
1. Knowledge と Policy の Source Path を Read ツールで開き、全文を取得する
2. それぞれの `##` セクションをすべて列挙し、各基準を変更差分・実行証跡・レポートと照合する
3. タスク要件を検証可能な最小単位に分解し、実装コードと現在有効な実行証跡で独立に確認する
4. 前段の raw finding と非指摘の懸念を再評価し、共通ヘルパー・正規化関数・builder・adapter の同等経路も照合する

**証跡の扱い:**
- 実コードに関する主張は、現在の review snapshot 上のコードを正本とします。
- 実行ログ・実行レポートは、その実行対象 snapshot が現在コードと一致する、または対象変更後に生成されたと確認できる場合だけ現在動作の証跡にします。`ビルド結果` / `テスト結果` も同じ条件を満たす場合だけ有効です。
- 古いレポートは探索ポインタに留めます。現在コードと矛盾する場合は再実行し、できなければ未確認として NEED_REPLAN にします。
- `検証証跡` は対象・確認内容・結果が揃う場合だけ補助証跡とします。モック、静的確認、限定的な単体テストは、その範囲を超える動作確認に使いません。
- 記録を探索できない、アクセスできない、または必要経路を探索し切れていない場合は未確認です。元要件または既存公開契約から存在・配線が必須と導け、必要な全経路を探索済みの場合だけ locationless issue にします。

**出力:** `supervisor-validation-finding-contract` 出力契約に従って記録してください。APPROVE は issue 0件かつ必須証跡を確認できた場合、REJECT は現在の観測欠陥 issue が1件以上の場合、NEED_REPLAN は issue化できない主要要件または必須証跡が未確認の場合にします。判定に関係なく `supervisor-gate-summary-finding-contract` 出力契約に従い、実際の判定、要点、次アクションまたは未完了理由を正しく要約してください。
