レビュー → 修正のループが {cycle_count} 回繰り返されました。

Report Directory 内の最新レビュー報告と修正報告を調べ、このループが健全（収束中）か、非生産的（発散・振動・停滞）かを判断してください。

**最初に観測時点を確定してください:**
- このモニターは、修正完了直後で修正後レビューがまだない時点にも発火します。直近のレビューより後に当該指摘を修正したとの報告があり、その後のレビュー証拠がない場合は「修正後の検証待ち」であり、反復や停滞ではありません。選択肢では健全・進捗ありの条件に含めてください。
- 当該指摘を修正したとの報告がない場合は、修正後レビューがないことだけで健全と判断しないでください。

**判断基準:**
- 同じ指摘が複数の修正後レビューでも継続しているか確認してください。
  - 修正済みとの報告があり、修正後レビュー待ち → 健全（通常の検証を続ける）
  - 修正後レビューが同じ指摘を再確認 → 非生産的な反復の候補
  - 以前の指摘が解消され、別の指摘が新しく出現 → 原則として健全
- 修正が実際にコードへ反映されているか、修正報告と現行コードを照合してください。
- 修正後レビューの指摘が現行コードと乖離している場合は、同一修正を繰り返さず、利用可能な再確認・立て直し・停止の選択肢から判断してください。
- 新しい指摘や再発した指摘の数が全体として減少しているか確認してください。

同一の問題が修正後レビューでも繰り返され、利用可能な打開手段がない場合に限り、停止を選んでください。
