専門レビュー後の累積差分がマージ可能な品質かレビューしてください。

手順:
1. Knowledge と Policy の Source Path が渡されている場合は開き、`##` セクションの基準を確認する
2. タスクの元要件、計画、変更差分、実行証跡、前段レビューを確認する
3. 要求未達、既存動作の破壊、テスト不足、要求外変更、明らかなリスクを探す
4. 差分で追加・変更された ID、名前、メタデータ、設定、環境変数、出力契約、型フィールド、status/discriminant、helper、adapter、entrypoint を列挙する
5. 列挙した項目ごとに、同種箇所、mock、fixture、factory、test double、保存・表示・実行入口を Grep / Glob / Read で確認し、確認した検索語とファイルを証跡に残す
6. 副作用や状態変更は、正常終了・早期終了・例外・中断・cleanup / rollback の経路を分けて確認する
7. `resolved` 判定は修正差分ではなく、元 finding の期待結果・受入条件・元要件へ戻って確認する
8. マージを止めるべき問題だけを、場所・影響・修正方針付きで指摘する
