再計画のループ（plan → テスト作成 → 実装 → レビュー → 修正 → plan）が {cycle_count} 回繰り返されました。

修正担当が「進められない」と申告するたびに plan へ戻る設計のため、このループ自体は異常ではありません。
判断すべきは、再計画が行き詰まりを解消する方向に働いているか、同じ行き詰まりを繰り返しているかです。

Finding Contract の ledger summary / `findings-ledger.json` がある場合は統合 ledger を一次情報として確認し、
ledger がない場合は Report Directory 内の最新の計画・レビューレポートを確認してください。

**最初に確認すること:**
このモニターはステップ名の並びだけで発火するため、直近の fix が「修正完了」で正常に終わった直後にも
呼ばれることがあります。その場合は行き詰まりではないので、レビューへの通常遷移を選んでください。

**判断基準:**
- 直近の計画が、前回の行き詰まりの理由に対して新しい分解・別のアプローチを示しているか
  - 計画の内容が毎回実質同じ → 非生産的（同じ壁に同じ助走でぶつかっている）
  - blocker の言い換えではなく作業の分解が変わっている → 健全
- 差し戻しの指摘（未解決 finding や REJECT 理由）が回を追って減っているか
- 修正担当の「進められない」理由が毎回同一文言か、具体化・前進しているか

非生産的と判断した場合のみ打ち切りを選んでください。打ち切りは人間への引き継ぎを意味するため、
残っている行き詰まりの内容と、試行済みのアプローチを要約して出力に含めてください。
